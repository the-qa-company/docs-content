# Requirements

To run QAnswer you need:

- a linux distribution (we use Ubuntu by default)
- and java 11 installed.

# Specs

For a graph of 100 million triples we reccomand a server with:

- 4 CPUs
- 8 GB of RAM
- 100 GB of SSD

# Installation budle

An on premise installation bundle contains 3 files:

- a jar with the current $version of QAnswer named QAnswer-$version.jar
- an application.properties file
- a README

You can ran QAnswer as:

 ```bash
java -jar QAnswer-$version.jar -Dspring.config.location=application.properties
```

and you will get:

<Image src='/img/newUI/premise/QAnswer-bash.png'/>

QAnswer will be available at [http://localhost:4567](http://localhost:4567).
The swagger documentation will be availabe at [http://localhost:4567/swagger-ui.html](http://localhost:4567/swagger-ui.html).

# Configuration

You can configure QAnswer by changing the application.properties file. The main properties are:

| Property      | Value | Default |
| :---        |    :----   |   :----   | 
| server.port      | port on which QAnswer will start | 4567 | 
| qa.public.url   | public url under which you expose QAnswer  | http://localhost:4567 | 
| qa.directory   | directory containing all data necessary to QAnswer | ./qanswer_data/ | 
| qa.admin.username | username of the admin account | admin |
| qa.admin.password | password of the admin account | 123456 | 
| spring.datasource.username| username of the database used in QAnswer | sa |
| spring.datasource.password | password of the database used in QAnswer | password | 
| app.jwtSecret| secret to generate the JWT token | MySuperSecret |

:::caution Warning
For security reasons in a production deployment you have to change the following properties:
- qa.admin.password
- spring.datasource.password
- app.jwtSecret
:::
